About BreathSmart

  • Brand Name: BreathSmart
  • Mission: To enhance environmental health awareness by providing real-time air quality monitoring.
  • Product Overview: BreathSmart offers an innovative solution to air quality monitoring through a compact and efficient embedded system. Utilizing the Arduino Nano 33 IoT, our system accurately measures various air quality parameters including temperature, humidity, carbon dioxide (CO2), and carbon monoxide (CO).
  • Technology and Components: Our system features state-of-the-art sensors integrated with the Arduino Nano 33 IoT to detect and record essential air quality metrics. Data transmission is handled securely and efficiently via Wi-Fi to an MQTT broker.
  • Data Management: The data collected is stored and managed using a robust combination of InfluxDB and Grafana, ensuring data integrity and ease of access for both real-time and historical analysis.
  • User Interface: BreathSmart provides an intuitive web interface displaying live air quality readings and historical data visualizations. This allows users to make informed decisions about their environment and health.
  • Communication Protocols: Leveraging MQTT for network communication and I2C/SPI for sensor interfacing, our system ensures reliable data flow and scalability.
  • Objective: With BreathSmart, our goal is to empower individuals, communities, and organizations to take proactive measures towards improving air quality in their environments.





Understanding How The Product Works

  • Sensors and Measurements: BreathSmart uses advanced sensors such as the DHT22 for temperature and humidity, CCS811 for CO2 levels, and a specific electrochemical sensor for monitoring carbon monoxide (CO). These sensors are chosen for their accuracy, reliability, and response times.
  • Data Acquisition: The Arduino Nano 33 IoT, equipped with built-in Wi-Fi, collects data from the sensors using digital communication protocols like I2C and SPI. Each sensors data is periodically sampled to provide an accurate representation of the environment's air quality.
  • Communication: Data from the Arduino is sent in real-time to an MQTT broker using the MQTT protocol. This protocol is chosen for its lightweight nature and its ability to provide reliable and efficient message delivery even in environments where network bandwidth is constrained.
  • Data Processing and Storage: Once the data is published to the MQTT broker, it is subsequently subscribed to by Telegraf, which acts as an intermediary to parse and store this data into InfluxDB, a time-series database optimized for handling massive amounts of timestamped data.
  • Real-time Data Visualization: Live data streaming is handled through a web interface that connects directly to the MQTT broker. Using modern web technologies such as JavaScript and WebSockets, the interface displays the data as it is received, enabling users to see real-time updates without needing to refresh the page.
  • Historical Data Analysis: Grafana is used to fetch historical data from InfluxDB, allowing users to visualize trends and patterns over time through a series of customizable dashboards. These visualizations help in making data-driven decisions and understanding long-term air quality trends.
  • System Scalability and Fault Tolerance: The architecture is designed to be scalable, supporting the addition of more sensors or integration into larger monitoring networks. Fault tolerance is achieved through rigorous testing of the system under various scenarios to ensure reliability in diverse conditions.
  • Environmental and User Interaction: The system is capable of responding to changes in environmental conditions by adjusting the frequency of readings or triggering alerts based on predefined thresholds, which can be customized according to user preferences.





FAQ's

  • What air quality parameters does BreathSmart monitor? BreathSmart measures temperature, humidity, carbon dioxide (CO2), and carbon monoxide (CO) using high-quality sensors to provide an accurate assessment of air quality.
  • How does BreathSmart send data in real-time? The system uses the MQTT protocol to send data from the Arduino Nano 33 IoT to an MQTT broker. This protocol is chosen for its efficiency and reliability in low-bandwidth environments.
  • Can I access historical data? Yes, historical data is accessible through the Grafana interface, which pulls data from InfluxDB. Grafana provides detailed visualizations of past air quality measurements.
  • Is the system difficult to set up? BreathSmart is designed for easy setup. Detailed setup instructions are provided in the user manual, which includes step-by-step guides on connecting sensors and configuring the software.
  • What should I do if my device stops sending data? Refer to the troubleshooting section below for common issues and solutions. Ensure that your device is connected to the internet and check the power supply.
  • How often are the sensors calibrated? We recommend calibrating sensors once every six months to maintain accuracy. Calibration procedures are outlined in the user manual.





Troubleshooting

  • Problem: Device not connecting to Wi-Fi. Solution: Ensure that your network settings are correctly configured and that your Wi-Fi network is operational. Restart the Arduino and try connecting again.
  • Problem: Data not appearing on the live data webpage. Solution: Check if the MQTT broker is online and functioning. Ensure that the web client is subscribed to the correct topic and that there are no network firewalls blocking the connection.
  • Problem: Sensors not reporting data. Solution: Verify that the sensors are correctly connected to the Arduino. Check for any visible damage to the sensors or wires. Replace the sensors if necessary.
  • Problem: Historical data not loading in Grafana. Solution: Ensure that InfluxDB is running and that Grafana is configured with the correct database credentials. Check the network connection between Grafana and InfluxDB.
  • Problem: Alerts not triggering. Solution: Confirm that the alert thresholds are correctly set in the system settings. Check the system logs for any error messages that could indicate why alerts are not functioning.
  • Problem: System reports frequent disconnections. Solution: This may be due to network instability or power interruptions. Ensure a stable power supply and consider using a UPS for continuous operation. Check the quality of your Wi-Fi signal and move the device closer to the router if necessary.
